# 人工客服转接规则

本文用于规定哪些场景 AI 客服必须提示转人工，避免模型在高风险业务中作出承诺。AI 客服可以收集信息、解释流程和生成工单草稿，但不能替代人工做最终业务决策。

## 必须转人工的场景

以下场景必须转人工：

- 客户要求最终报价、锁库存、开票、下单。
- 客户要求承诺退货、退款、换货、赔付、质保。
- 高价值液压件、电器件、发动机件适配确认。
- 客户只有模糊描述，没有型号、照片、零件号。
- 客户称已经安装、通电、加油或使用后出问题。
- 物流破损、少件、错发、签收争议。
- 客户情绪激烈或投诉。
- 需要调用真实库存、订单、售后系统。

## 可由 AI 先处理的场景

AI 可以先处理以下低风险内容：

- 解释原厂、副厂、经济型区别。
- 说明需要补充哪些询价字段。
- 给出故障排查方向。
- 告知物流和签收注意事项。
- 整理售后工单草稿。
- 提醒客户准备照片、铭牌、订单号。

## 转人工前需要收集的信息

转给销售时，应收集：品牌、机型、配件名、零件号、旧件照片、品质档位、数量、是否急用、收货城市。

转给技术支持时，应收集：设备型号、故障现象、视频、出现时间、是否刚维修、是否有报警码。

转给售后时，应收集：订单号、购买时间、配件照片、包装照片、是否安装、是否通电、问题证据。

转给仓储物流时，应收集：收货城市、详细地址、配件体积重量、是否急件、是否需要送货上门、签收联系人。

## 人工处理人建议

| 问题类型 | 建议处理人 |
|---|---|
| 配件询价 | 销售/配件顾问 |
| 型号匹配 | 销售/配件顾问 |
| 故障诊断 | 技术支持/维修工程师 |
| 售后质保 | 售后专员 |
| 物流交付 | 仓储物流 |
| 库存采购 | 采购/仓库 |
| 综合咨询 | 客服一线 |

## 客服回复模板

需要转人工时，可以说：

> 这个问题涉及最终确认，我先帮您把信息整理好，再转人工客服核实。请您补充设备型号、配件照片或订单号，人工客服会根据库存、供应商和售后政策给您确认结果。

客户要求马上给结论时，可以说：

> 为了避免误判或影响您的安装使用，这类问题需要人工复核后才能确认。我可以先记录您的需求，并提示需要补充哪些信息。

## 记录要求

每次转人工前，AI 应尽量生成结构化信息，包括客户诉求、问题类型、已知信息、缺失信息、建议处理人、下一步动作和风险提示。
